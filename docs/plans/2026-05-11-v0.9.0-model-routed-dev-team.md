# v0.9.0 — Model-routed dev team (implementation plan)

**Spec:** `docs/architecture/2026-05-11-v0.9.0-destination.md`
**Codex thread:** `019e1536-0211-7d50-a307-d16d092d44b7`
**Architecture rounds:** 7 (all SHIP'd — see sidecar)
**Target release:** v0.9.0 (Codex + Claude-task + Ollama adapters; Gemini deferred to v0.9.1; Grok deferred to v0.9.x)

## Goal (one paragraph)

Implement the L11 discipline layer over CLI-routed dev-team dispatch. Replace the v0.8.x assumption that experts are single-model Claude Task subagents with a routing-aware dispatcher that picks the best-suited CLI per role from a preference ladder. Add Ollama Cloud as a second CLI adapter alongside Codex. Wire the existing 5 codex-paired skills to dispatch experts at appropriate phase boundaries. Make TDD-mandatory with panel-mode expert-test review. Make ralph-loop halt-aware. Preserve the L11 verdict protocol, sidecar audit, mailbox primitive, and double-SHIP discipline across both runtime kinds (Claude-task and CLI-harness).

## In scope (v0.9.0)

- `lib/codex-bridge/cli-harness/` Node module — spawn-based dispatcher for non-Claude CLIs
- `cli-adapters/codex.js` + `cli-adapters/ollama.js` — vendor adapters with 4-path test gate
- `role-routing.json` + `role-recommendations.json` — preference-ladder configuration surface
- Doctor extension — availability detection + 1h TTL cache + smart invalidation
- Panel-mode dispatcher — multi-CLI consensus with deterministic verdict aggregation
- Sidecar schema migration — `codex_session` → `role_sessions`, role-prompts versioning, resolution audit
- Skill prose updates — experts wired into brainstorming/writing-plans/TDD/SDD/systematic-debugging
- TDD-mandatory in writing-plans (panel-mode `expert-test`)
- Ralph-loop halt-aware (terminal halt envelope)
- Test strategy implementation — Tier 1–3 + 6 default CI; Tier 7 release gate

## Out of scope (v0.9.1+)

- Gemini adapter (v0.9.1 — same admission gate)
- Grok adapter (v0.9.x+ — no first-party CLI per xAI docs)
- Qwen / DeepSeek adapters (v0.9.x+)
- Local Ollama path (`ANTHROPIC_BASE_URL=localhost:11434`)
- Session continuity for any adapter (v0.9.1+)
- Cross-CLI replay (recorded but not actively exercised)
- Plugin rename to `paired-superpowers` (v1.0.0)
- Composer-driven `high_stakes` detection (v0.9.x — currently user-controlled)
