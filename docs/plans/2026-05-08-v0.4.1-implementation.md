### Audit-trail preservation sequencing (resolves Slice 4/5 inconsistency)

The intent: gitignore the hidden dir by default while preserving already-tracked audit trails via `git mv` (which keeps the file tracked at the new path; `.gitignore` does not untrack already-tracked files).

**Order matters.** The plan executes:

1. **Slice 4** (this comes first): the migration script runs. Tracked sidecars use `git mv` (history preserved). Untracked sidecars use plain `mv` and remain untracked at the new path. The migration commit captures `git mv` renames + script. Untracked files don't get auto-staged.
2. **Slice 5**: add `.superpowers-codex-paired/` to `.gitignore`. Already-tracked files (the migrated `validation-rubric` audit) STAY tracked because `.gitignore` only affects untracked files. No `-f` needed for already-tracked files. Untracked sidecars (autopilot/v0.4.1 review trails) remain untracked AND ignored, which is the desired state.

So `git add -f` is NOT required for the validation-rubric audit. It's already tracked. The plan body is corrected accordingly; the `-f` references in earlier drafts are removed.

## Slice 4: Migration script

**Validation:** critical (irreversible disk operations)

**Files:** create `scripts/migrate-sidecars-to-hidden-dir.sh`.

### Tasks

- [ ] **Step 4a: Create the migration script** per the spec § "Migration of existing sidecars" state machine. **Two-phase design** (preflight + execute) — never partial-mutate on ambiguous state:

  **Phase 1: preflight.** Walk every candidate; classify each into one of these states without mutating anything:
  - `migrate-tracked` (legacy=exists,tracked + hidden=absent)
  - `migrate-untracked` (legacy=exists,untracked + hidden=absent)
  - `already-migrated` (legacy=absent + hidden=exists)
  - `ambiguous` (legacy=exists + hidden=exists, regardless of tracked status)

  **Phase 1 exit conditions:**
  - If any candidate is `ambiguous`: print all conflicting paths + a remediation hint, exit nonzero with `0 mutated` so the user can resolve manually before re-running. **No file is moved.**
  - If `--dry-run`: print all classified actions and exit 0 without mutating.
  - Otherwise: proceed to Phase 2.

  **Phase 2: execute.** Walk the preflight classifications:
  - `migrate-tracked`: `mkdir -p` hidden parent + `git mv` source → hidden.
  - `migrate-untracked`: `mkdir -p` hidden parent + plain `mv` source → hidden.
  - `already-migrated`: no-op.
  - Print per-file action.

  This matches the spec's "halt on ambiguous state rather than guessing" rule.

  **Mutation safety:** the script **never mutates if Phase 1 detects ambiguity** (exit nonzero, 0 moves performed). However, Phase 2 itself is not transactional — if a single `git mv` or `mv` fails partway through (e.g., disk full, permission denied on a specific path), prior moves in the same Phase 2 run will already be committed. Recovery: re-run the script. The state machine is idempotent for `migrate-tracked` and `migrate-untracked` cases that have already completed (they become `already-migrated` on the next run); the failed case will re-attempt. If a Phase 2 partial failure produces an `ambiguous` state on the next run (because the user manually intervened), the next preflight will halt and surface the conflict for manual resolution. Document this in the script's header comment so re-runs are predictable.

- [ ] **Step 4b: Make executable**: `chmod +x scripts/migrate-sidecars-to-hidden-dir.sh`.

- [ ] **Step 4c: Write fixture-based shell tests at `tests/scripts/migrate-sidecars.test.sh`** covering the state machine. Each test creates a throwaway repo (`mktemp -d`, `git init`), seeds the relevant fixture state, runs the migration script in `--dry-run` or real mode, and asserts the outcome. Required cases (the script will be called with `--dry-run` for classification-only assertions; without for execution assertions):
  1. **`migrate-tracked` classification**: throwaway repo with one tracked legacy sidecar (`git add` + commit a `*.codex.json`); run `--dry-run`; assert stdout mentions `migrate-tracked` for that file; assert exit 0; assert no file moved.
