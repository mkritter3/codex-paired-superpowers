## Goal

Wire write-capable implementer agents into the v0.9.0 dispatcher. Multiple agents (any combination of Codex CLI, Claude CLI pointed at Anthropic or at Ollama Cloud via `https://ollama.com/v1/messages`) implement in parallel within one slice. They communicate via the existing mailbox primitive at execution checkpoints (not mid-token). Result merged via integration branch, paired-reviewed by Codex+Claude, with merger-agent path for conflicts (also requiring double-SHIP).

10 slices, opt-in via `**Implementers:**` frontmatter, no v0.9.x regression.

**DependsOn:** slices 3, 4, 5.

### Tasks

1. `mailbox-poller.js`: export `createMailboxPoller({memberId, repoRoot, sliceId, implementerRunId, cadenceMs = 45_000, jitterMs = 10_000, clockNow = Date.now, scheduler = setTimeout, clearScheduler = clearTimeout})`. Returns `{pollNow, start, stop, on(event, handler)}`. The `clearScheduler` seam is injectable so `stop()` cleanup tests don't depend on global `clearTimeout` (round-2 reviewer fix).
2. `pollNow()` reads `readUnreadMessages(repoRoot, memberId)` non-blocking. Emits `delivered` event per new message.
3. `start()` schedules timed cadence with random jitter in `[-jitterMs, +jitterMs]`. Idempotent.
4. `stop()` clears scheduler. Idempotent.
5. Cadence calculation injects via `scheduler` (default `setTimeout`) — allows fake-timer test injection.
6. Jitter source injects via a deps function — allows deterministic test injection.
7. Each `delivered` event records a `mailbox_delivered` sidecar event via `appendImplementerEventLocked`.
8. Each `pollNow` invocation records a `mailbox_poll` sidecar event (debounce: only if it's been >1s since last poll-event, to avoid log flood under busy checkpoint patterns).
9. `mailbox.js`: extend the existing message validator to accept optional fields per spec: `kind`, `priority`, `implementer_run_id`, `slice_id`, `body_hash`. Old messages without these fields still validate.
10. `mailbox-poller.js`: export `injectMailboxDelivery(adapter, messages)` — adapter-specific injection (Claude CLI: MCP mailbox tool when available; otherwise prompt-injection at next continuation; Codex CLI: workspace-local mailbox command shim).
