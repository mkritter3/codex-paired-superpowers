The scenario runner does NOT own native `/computer-use`. Per spec, Claude in the active Claude Code session drives `/computer-use` foreground; the runner exposes an injectable adapter so tests can substitute mocks AND real implementation can hook in at slice 11 (autopilot SKILL.md).
