# v0.7.0 Implementer Routing — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (or this plugin's forked equivalent). The autopilot we are extending is the same autopilot driving execution; Phase B routing itself is what we are building, so we cannot self-drive Phase B for this plan. Steps use checkbox (`- [ ]`) syntax.

**Spec:** `docs/specs/2026-05-08-v0.7.0-implementer-routing.md`

**Goal:** Implement v0.7.0's implementer routing — Codex-default + Sonnet-fallback dispatch per slice, parallel slice execution in symlink-bootstrapped git worktrees, ordered cherry-pick integration, subject-only commit conventions.

**Architecture:** Twelve sequential slices: parsers (1–2) → config (3) → worktree primitives (4) → sidecar (5) → dispatch wrappers (6–7) → routing (8) → integration (9) → hook (10) → orchestrator wiring (11) → smoke + release (12). Each slice is independently testable. The v0.6.0 autopilot drives this plan in serial; once v0.7.0 ships, future plans get parallel slice dispatch.

**Tech Stack:** Node.js 20+ (existing bridge code; zero npm deps). Bash for hook + smoke tests. JSON only for `.codex-paired/project.json` (continuation of v0.6.0 decision). All git shell-outs use `node:child_process.execFileSync` with argv arrays (no shell interpolation, no injection risk) — same convention as the existing bridge modules.

## Slice 2: Conflict detection

**Files:**
- lib/codex-bridge/parallel-detect.js
- tests/codex-bridge/parallel-detect.test.js

**Rationale:** This is the gate that decides whether autopilot dispatches in parallel or serial. Bugs here cause silent serialization OR incorrect parallel dispatch with file conflicts.

### Tasks

- [ ] **2a: Test list.** For `detectParallelBatches(planPath, sliceIds)`:
  - 3 consecutive slices, all with non-overlapping Files → 1 batch with `mode:"parallel"` containing all 3
  - 3 consecutive slices, slices 2+3 share a file → 2 batches: `[parallel:[1]]` then `[serial:[2,3]]`
  - 1-slice batches always return `mode:"serial"` for clarity. Multi-slice batches return `mode:"parallel"` only if all pairs are non-overlapping; otherwise `mode:"serial"` for the whole batch.
  - **Halt-in-window semantics:** When a halt occurs for a slice in the current candidate window, the detector closes the window IMMEDIATELY (no batch returned for the partial window) and emits the halt entry. Subsequent slices restart batching in a fresh window. This guarantees autopilot sees the halt before any worktree setup is attempted for the affected window.
  - Slice missing Files block but in parallel candidate window → halt entry `{slice_id, halt:"parallel-files-missing"}`; current window closes; later slices may form new batches
  - Slice with malformed Files → halt entry `{slice_id, halt:"parallel-files-malformed"}`; current window closes
  - **Autopilot integration contract (slice 11):** if `halts[]` is non-empty, autopilot MUST halt before initiating worktree setup. The detector returning `{batches, halts}` is data; the orchestrator enforces stop-before-dispatch. Test this contract via slice 11's structural smoke.
  - Implementer directive ignored — only Files matter for conflict
  - Mixed implementer (codex + sonnet) with non-overlapping Files → still parallel
  - Returns `{batches: [...], halts: [...]}` shape

- [ ] **2c: Implement `lib/codex-bridge/parallel-detect.js`.** Single export `detectParallelBatches(planPath, sliceIds: string[]) → {batches, halts}`. Uses `parseFilesBlock` from slice 1. Algorithm: iterate slice_ids in order; collect contiguous non-overlapping run into a batch; on overlap, close current batch as serial, start new batch with the conflicting slice.

## Slice 4: Worktree primitives

**Files:**
- lib/codex-bridge/worktree.js
- tests/codex-bridge/worktree.test.js

**Rationale:** Owns git-worktree lifecycle. All git operations use `node:child_process.execFileSync` with argv arrays (no shell, no injection). Tests use real temp git repos.

### Tasks

- [ ] **4a: Test list.**
  - `create(repoRoot, sliceId, sliceStartSha)` creates `<repo>/.git-worktrees/slice-<N>` branched as `<slice-id>-impl` from `slice_start_sha`. Returns absolute worktree path.
  - `create` halts `worktree-gitignore-missing` if `.git-worktrees/` not in repo `.gitignore`
  - `create` halts `worktree-path-conflict` if path exists and is not a clean worktree of the same slice
  - `create` halts `worktree-create-failed` on `git worktree add` nonzero exit (with stderr surfaced)
  - **Bootstrap symlink policy (default vs configured):**
    - **Default candidates** (`node_modules`, `.venv`, `venv`): `bootstrap` calls these "auto-detect" — try to symlink if the source exists in `repoRoot`; skip silently (no halt) if the source is absent. This means a Node-only repo with no `.venv` does not halt because `.venv` is a default candidate.
    - **User-configured entries** (anything in `worktree_bootstrap.symlinks` that is NOT one of the three defaults): MUST exist in `repoRoot`. If absent → halt `worktree-bootstrap-failed`. The author opted in explicitly, so absence is a real defect.
    - The `symlinks` argument to `bootstrap` carries metadata distinguishing the two: `[{path, required: boolean}]`. Project-config (slice 3) tags defaults as `required:false` and user entries as `required:true`.
  - `bootstrap` halts `worktree-bootstrap-failed` if symlink creation fails (e.g., permission error) OR if target already exists and is not the correct symlink
  - `bootstrap` is idempotent: re-running over an already-correctly-symlinked worktree succeeds without rework. Idempotency check: `lstat` target; if it's a symlink pointing at the expected source, treat as success.
  - **`verifyBootstrap(worktreePath, recordedSymlinks)`** (new helper, used by orchestrator slice 11 as Tier 2 dispatch gate). For each entry in `recordedSymlinks`:
    - `lstat(worktreePath/<symlink>)` — must be a symlink (else `{ok:false, failed:[{symlink, reason:"missing"|"not-a-symlink"}]}`)
    - `readlink(worktreePath/<symlink>)` resolves to the expected absolute path (else `{ok:false, failed:[{symlink, reason:"wrong-target", expected, actual}]}`)
    - Returns `{ok:true}` only if every recorded symlink verifies. Halt code surfaced by orchestrator is `worktree-bootstrap-stale`.
  - Tests for `verifyBootstrap`: passes on freshly bootstrapped worktree; fails with `missing` when symlink deleted; fails with `wrong-target` when symlink swapped to a different path; fails with `not-a-symlink` when path becomes a regular file.
  - `reset(worktreePath, sliceStartSha)` runs `git reset --hard <sha>`; preserves untracked symlinks; halts `worktree-reset-failed` on nonzero exit
  - `remove(repoRoot, worktreePath)` runs `git worktree remove`; halts `worktree-cleanup-failed` on nonzero exit
  - `removeBranch(repoRoot, branchName)` runs `git branch -D`; halts `worktree-branch-cleanup-failed` on nonzero exit
  - All halt returns shape `{ok:false, halt:{reason, detail}}`; all success returns `{ok:true, ...payload}`
