## 1. ~~Migrate Sonnet dispatch to agent teams~~ — RESOLVED 2026-05-09

**Original concern:** I had a hand-wavy claim that `Task` subagent dispatches serialize at the runtime layer. That claim was based on an early Claude Code docs review that listed Task parallelism as "undocumented", combined with an unverified assumption that the MCP serialization observed for codex generalizes.

**Empirical test (2026-05-09):** dispatched two `Task` subagents in a single assistant turn, each running `sleep 5`. Result: A finished at T=14s post-dispatch, B at T=15s — gap of 1 second. If serialized, B would have started after A's completion (~T=19s+). They ran concurrently. Per-agent wall-clock includes ~9s of subagent spin-up overhead, but the actual work overlapped fully.

**Conclusion:** `Task` parallelizes natively when calls are issued in a single assistant turn. v0.7.1's Sonnet-side parallelism claim is valid as written. No agent teams migration needed for parallelism reasons.

**What still might motivate agent teams later:** isolated context windows per teammate, mailbox-style coordination, separate API quotas per teammate. None of these are urgent.

**Reference:** `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` documented at https://code.claude.com/docs/en/agent-teams; Claude Code v2.1.32+.

## 2. Domain-aware routing policy

**Problem:** v0.7.0's routing only considers `**Implementer:**` directive (codex / sonnet). It does not enforce a domain-based safety policy.

**User policy:**
- UI / UX work → ALWAYS Claude (Codex has weak UI/UX judgment)
- AI harness work (autopilot orchestrator, hooks, MCP servers, Codex itself) → ALWAYS Claude (Codex shouldn't write the systems that govern it)
- Backend / pure logic / data plumbing → Codex acceptable
- Mixed / unclear → Claude (default safe)

**Solution:** SKILL.md pre-dispatch checklist gains a domain check:
- Plan frontmatter optionally declares `**Domain:** ui | ai-harness | backend | general` per slice.
- If `Domain: ui` or `Domain: ai-harness`, force Sonnet/Claude regardless of `**Implementer:**` value.
- If `Domain` is absent, Claude inspects the slice's `**Files:**` paths and infers: paths under `web/`, `app/`, `src/components/`, `*.tsx`, `*.css`, `*.html`, `frontend/` → ui. Paths under `skills/`, `agents/`, `hooks/`, `lib/codex-bridge/`, MCP server code → ai-harness. Otherwise general/backend.
- The orchestrator halts with `domain-policy-violation` if the explicit `**Implementer:** codex` clashes with an inferred or declared `ui` / `ai-harness` domain. User must fix the plan.

**Default flip:** consider whether to flip the default from Codex to Claude. v0.7.0 defaults to Codex on absent directive. Given the policy ("Codex okay for backend, Claude for everything else"), Claude is the safer default. Plan author opts into Codex per slice for backend work.
